Hexo+GitHub=Blog

Posted on 2019-01-08 | Edited on 2019-06-03 | Comments: | Views:

记录一下把github pages整成博客的过程。学东西最多最扎实的过程好像是大四->研三下，部分原因是由于认真做笔记，后来专心搞科研（为发文章求毕业所累），现学现用马上就忘记，除了写了点代码跑了点流程，有点想不起来自己会啥。因而，刚好刚毕业博后还没搬砖比较有空，开始捡起这个习惯。

1. 安装Node.js

下载Node.js安装，命令行node -v 和npm -v检查安装好，这里的npm类似于Python里的pip。或者mac下home brew安装node和npm。

2. 安装使用Hexo

Hexo是博客框架。建立Blog目录，用于存放Hexo框架和后面的博客网页文件，进入Blog目录，执行Hexo安装命令。 

# 初始化项目
npm init -f  
# 更换成淘宝的源会更快一些
npm config set registry https://registry.npm.taobao.org
# 安装hexo, 可以加-g选项设定为全局安装，然而如果不是用brew安装node和npm的话会有权限问题。
sudo pm install hexo-cli -g

Hexo的具体使用可见其文档。初始化博客。 

hexo init blog
cd blog 
npm install

建立本地测试站点。

# 新建测试站点
hexo new test_site
# 生成静态网页  
hexo generate
# 启动本地服务预览 
hexo server

3.关联本地Hexo和远端Github

在之前的blog目录下编辑站点配置文件_config.yml的最后几行，如下：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
    type: git
    repo: https://github.com/YaqiangCao/yaqiangcao.github.io.git
    branch: master

然后再执行命令执行设置：

npm install hexo-deployer-git --save

将本地博客更新到GitHub，以后写博客和更新基本就是这几行命令了，加上一开始的hexo new：

hexo clean
hexo generate 
hexo deploy

就可以围观新博客了，应该不至于像之前用过的点点突然就关了，或者WizNote付费后居然有些笔记丢失了（再也不想给它付费了）。下面就是各种折腾改配置改主题以及让它支持显示jupyter-notebook文件了。

4.更换主题

在Hexo Themes 随便选一个就可以了。按照说明安装，我选了一个比较简单不用太花心思打理的NexT。
