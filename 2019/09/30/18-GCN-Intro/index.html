Bui T D, Ravi S, Ramavajjala V. Neural graph learning: Training neural networks using graphs[C]//Proceedings of the Eleventh ACM International Conference on Web Search and Data Mining. ACM, 2018: 64-71.

why read it

看到了tensorflow的新装备Neural Structured Learning, 刚好前一阵和基友们聊天的时候听说图网络挺有意思的，就看看换换调参疲惫的心，看完觉得真的挺有意思的。

损失函数

基本上基于Label Propagation魔改成cost函数,问题的本质是使用已有标签数据和未知标签数据的关系，给未知标签数据打标签。 令一张图G=(V,E,W), V代表节点，E代表边，W代表边的权重，也可以简单操作成邻接矩阵, 用V_l代表已有label的节点，V_u代表没有label的节点。公式里LL代都有标签的边的集合，LU代表一个结点有标签另一个没有的边的集合，UU代表那些没有标签的边，d()是距离函数(l1或者l2距离)，w代表训练后的权重，h()代表训练后的隐藏层信息（或者理解成embeding features也成吧），a是超参权重, c()则是输如x和输出y的cost函数, g()是训练x到y的函数，一通操作猛如虎，再看还是深度学习练权重。



网络输入与结构

看文章前就一直在想这种图网络该怎么把网络信息融入进去参与训练, 文章提供了三种思路:

  1. using generic graphs such as Knowledge Bases, that consists of relationship links between entities; 怼到生物数据上，就是例如已知的protein-protein interaction，或者是3D genome interaction, 最直观合理的操作。
  2. using embeddings learnt by an unsupervised learning technique; 玄学表示，怼到生物学数据上，可能是一堆单细胞数据在gene/genomic features层面PCA或者t-SNE降纬, 然后就再用距离信息去衡量？
  3. using sparse feature representations for each vertex; 这点没明白他们想表达啥。

具体到代码实现层面怎么把边的信息输入模型，还是看到文章中的示意图才明白的，就是把例如邻接矩阵A，把i行取出来当作i结点的网络输入。
